Android 跨进程通信

发表于 2019-08-18 更新于 2021-03-12 分类于 技术积累

做 Android 开发，必然会遇到跨进程通信问题，网上有各种各样的总结，那针对我自己的理解，下面我将对 Android 的跨进程通信、进程内通信方面做一些总结，主要内容如下。

Android 跨进程通信

一、基础知识简介

在介绍 Android 跨进程通信之前，先简单介绍下进程隔离、进程通信。

1，进程隔离
在操作系统里，进程和进程之间的内存和数据都是不共享的。两个进程就像两个国家，各自为政。目的是为了避免进程间互相操作数据，从而引发安全问题。所以为了实现隔离，进程分别在不同的虚拟地址里，以此实现隔离。
2，跨进程通信
扩展上面的例子，两个国家虽然各自为政，但是会有互通的时候。那进程也是一样的，功能有限，想要协作，那就免不了跨进程去通信。这时候就需要采用特殊的通信机制：进程间通信（IPC:Inter-Process Communication，即进程间通信或跨进程通信，后文以IPC替代，在此声明）。

四、Android进程内通信

前面总结了几种 Android 跨进程通信方式，那也介绍下 Android 进程内部组件间的通信方式。

startActivity() 方式

显示调用 startActivity()。可以用 Intent 携带数据，而且如果用 startActivityForResult()，还可以得到目标 Activity 完成任务后的反馈。

startService、bindService 方式

用于与 Service 通信，其中 Intent、回调方法等会携带数据或者对象，选择哪一个需要视情况而定。

Boradcast 方式

对于全局广播，前面已经吐槽很多了，能不用则不用。但还有一种局部广播 LocalBroadcastManager，只能在 App 内部使用，也是一种进程内通信的方法，且在性能、安全等方面都要优于全局广播。

Handler 方式

这种方式在子线程和主线程通信中用得很普遍，跨模块使用也常见到，使用方便。需要注意的是Looper和线程问题。

五、Binder概述

Binder 是 Android 框架中非常重要的一个机制，在 Framework 中被广泛使用，理解 Binder 对阅读 Framework 源码有着很大的帮助。当然，在 Android 应用层的跨进程通信中，也被广泛使用。

Binder是什么？

Binder 英文单词的意思就是 “粘合剂”，把两个物体粘合在一起。说起 Binder 是什么，在 IPC 过程中，从不同的角度来看，它可以被定义为一种机制，实体类、远程代理或者传输对象，这里说的 Binder 是从宏观定义来看的，是说的 Binder 机制。在 Android 中，它是一种高效的 IPC 方式，是 Android 所特有的机制。它采用 C/S 架构模式，基于内存映射（mmap（）），在系统内核空间将 Client 端和 Service 端两个用户空间的进程联系在一起。

Binder的由来

Binder 前身是某公司开发的 OpenBinder 项目，后来该项目的作者加入了 Google，同时把该项目带进了 Android。从此以后，Binder就成为了 Android 中主要的跨进程通信机制。
