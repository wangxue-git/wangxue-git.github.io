进程和线程到底是什么关系

发表于 2020-08-18 更新于 2020-08-24 分类于 技术的“道”

很多程序员，可能至今也没弄懂进程和线程。本篇文章目的就是以通俗易懂的语言，让有一定计算机基础（至少需要知道 CPU 是什么）的人，明白进程和线程到底是什么，到底有什么关系。

进程的官方定义？

维基百科：进程（英语：process），是指计算机中已运行的程序。
百度百科：进程，进程是一个具有一定独立功能的程序关于某个数据集合的一次运行活动。

线程的官方定义？

维基百科：线程（英语：thread）是操作系统能够进行运算调度的最小单位。
百度百科：线程（英语：thread）是操作系统能够进行运算调度的最小单位。

小结一下，仅看官方定义，你一定还是一头雾水，不明白进程和线程到底是什么。但是，官方的定义是非常重要的。学习一个新概念之前，一定要先看官方定义。等你学完这个概念，理解了之后，再回头看官方的定义，你就会发现官方的概念是简洁而准确的。

进程，线程的通俗解释

进程比较好理解，就是程序一运行，我们就可以说程序创建了一个进程（至少一个，多进程不在本篇的讨论范围之内）。当程序运行结束，该进程就会被销毁。所以我们也可以说，程序是永久存在，进程只是暂时存在。

进程的英文单词是 process，process 还有一个意思，就是处理。计算机是一个新事务，在进程的概念出现之前，process 这个单词就已经存在了，那么为什么要把进程命名为处理呢？

这就和进程的本质有关了，程序每次的运行，本质上就是发出些指令，需要 CPU 来执行，站在 CPU 的角度来看，可不就是在处理吗，处理程序发出的指令。所以把进程命名为 process，是非常形象的，也能够帮助接触到的人理解进程的本质。

对进程这个概念的理解过程，也能够体现阅读英文文档的重要性，尤其是计算机领域。计算中很多概念都是来自英文世界，如果只看中文文档，很多概念是意译。就是说翻译者是根据自己的理解，然后给翻译成中文，在很多场景下缺少了英文描述的准确性，人为增加了接触者的理解难度。

对计算机从业者来讲，学会阅读英文文档，不仅可以让你学习到最新的技术，更能帮助你准确理解概念，降低学习难度。
我们知道，CPU 的计算资源是独占性的，每次只能处理一个任务（此处为了理解方便，只考虑单核 CPU 的情况，多核 CPU 可以理解为多个 CPU）。那么，如果进程内的一个任务阻塞了，其他任务不就全阻塞了吗？如果按照这样的方式设计，计算机的运行效率就会特别低，也没有充分利用好 CPU 的硬件资源。

为了提高计算机的运行效率，充分利用好硬件资源，线程的概念便孕育而生。把一个进程内要处理的任务细分，以任务为单位发送给 CPU，CPU 以每个细分好的任务为单位来处理，如果一个任务阻塞了（比如等待输入），CPU 就去处理其他任务，直到原来的任务有响应了，CPU 才会再转过头来处理之前的任务（CPU 的实际过程调度过程会更复杂，任务还会有优先级之分）。这里的每个小任务，就是指线程了。

由此，我们可以知道，线程是为了充分利用好 CPU 而从一个进程内分出来的。也就是说，一个线程必从属于某个进程；一个进程可以有多个线程。

线程的三种实现方式

我们知道，先有了进程，然后为了加快处理速度，更好地利用 CPU 资源，才有了线程。刚开始时，用户有并发处理的需要，但是操作系统并不支持多线程，所以只能由软件工程师利用函数库，在代码里面手动创建线程，然后再以线程为单位，传给 CPU，提高程序的处理。这种由用户自己创建线程的方式，就叫做用户线程（User Thread）。

用户线程

随着用户线程的普及，各个操作系统(Windows,Solaris,Linux)内核也逐渐支持了多线程。从此用户不需要再在代码里面手动创建线程，操作系统会自动将进程处理成多线程的执行，实现并行操作。这种由操作系统内核自动创建线程的方式，就叫做内核线程（Kernel Thread）。

内核线程

在内核线程实现后，有操作系统(Solaris,Linux)为了能够更高效的运行，使用了轻量级进程（Light-weight process）方案。所谓轻量级进程，首先在操作系统内核中支持用户线程，并且一个进程可以有多个轻量级进程/轻权进程，每个轻权进程再由一个单独的内核线程来支持。但是，由于这种机制过于复杂，在 Solaris 的实际实践中并没有发挥出想象中的优势，所以 Solaris 就又退回了内核线程方案。

轻量级进程

总结

程序运行时，会至少创建一个进程，进程又会创建从属的线程。不同的进程间相互隔离，同一个进程内的所有线程共享内存，其中一个线程崩溃，则该进程内的线程全部奔溃。一个线程必从属于某个进程；一个进程可以有多个线程。

js vue node koa 这些技术之间的关系

发表于 2020-08-16 更新于 2020-08-24 分类于 技术的“道”

许多初学者，或者想走向全栈的前端开发者，对 node 和 koa 的关系认识混乱，阻碍了大家技术的进步，所以我打算简述一下这几个技术之间的关系，希望能够对初学者有所帮助。

先声明，此文章仅为了让初学者能够了解 js vue node koa 这些技术之间的关系，对每项技术的深入讨论，并不在本篇文章的写作目的之内，敬请谅解。

js是什么？

JavaScript，简称 JS，是一种高级脚本语言（编程语言），在前端开发中具有绝对的统治地位。

JavaScript 在1995年由 Netscape 公司的 Brendan Eich，在网景浏览器上首次设计实现而成，用来控制浏览器行为，让网页能够实现更复杂的功能。因为 Netscape 公司与 Sun 公司（创造 Java 语言的公司）合作，Netscape 管理层希望它外观看起来像 Java，因此取名为 JavaScript。

1996年8月，微软模仿 JavaScript 开发了一种相近的语言，取名为 JScript（JavaScript 是 Netscape 的注册商标，微软不能用），首先内置于IE 3.0。Netscape 公司面临丧失浏览器脚本语言的主导权的局面。

1996年11月，Netscape 公司决定将 JavaScript 提交给国际标准化组织 ECMA（European Computer Manufacturers Association），希望 JavaScript 能够成为国际标准，以此抵抗微软。ECMA 的39号技术委员会（Technical Committee 39）负责制定和审核这个标准，成员由业内的大公司派出的工程师组成，目前共25个人。该委员会定期开会，所有的邮件讨论和会议记录，都是公开的。

1997年7月，ECMA 组织发布262号标准文件（ECMA-262）的第一版，规定了浏览器脚本语言的标准，并将这种语言称为 ECMAScript。这个版本就是 ECMAScript 1.0 版。

之所以不叫 JavaScript，一方面是由于商标的关系，Java 是 Sun 公司的商标，根据一份授权协议，只有 Netscape 公司可以合法地使用 JavaScript 这个名字，且 JavaScript 已经被 Netscape 公司注册为商标，另一方面也是想体现这门语言的制定者是 ECMA，不是 Netscape，这样有利于保证这门语言的开放性和中立性。

有了 ECMA 组织的加持，JavaScript 很快地统治了众多浏览器，成为了前端开发中最流行的脚本语言。

ECMAscript 5.1 版在2011年6月发布之后，ECMA 就使用了新的命名方式。2015年6月，ECMAScript 6 正式发布，并且更名为“ECMAScript 2015”。之后每年发布一个 ECMAScirpt 的版本，分别为“ECMAScript 2016”，“ECMAScript 2017”等。

但是，很多人还是习惯于使用 ES6 来称呼 ECMAscript 5.1 版之后的版本。所以，如果你听别人说 ES6，其实并非特指“ECMAScript 2015”这一个版本，而是指 ECMAscript 5.1 版之后所有的版本。

总结一下，JavaScript，简称 JS，是前端开发中最流行的脚本语言。JavaScript 的标准由国际标准化组织 ECMA 制定，老版本一般被称为 ES5，具有新特性的版本被称为 ES6。并且如果没有特殊说明，ES6，并非特指“ECMAScript 2015”这一个版本，而是指ECMAscript 5.1 版之后所有的版本。

vue是什么？

Vue (读音 /vjuː/，类似于 view) 是一套用于构建用户界面的渐进式框架，全称 Vue.js。通俗的讲，vue 是一个前端框架，使用vue 进行开发，可以大大加快前端的开发速度。

node是什么？

node 全称 Node.js，是一个基于 JavaScript 语法的服务器端开发语言（类似于JAVA）。Node.js 基于 JavaScript 语言，并且整合了谷歌的 V8 引擎，是一种轻量的、可模块化的开发工具。

Node.js 可以用于 Web 后端服务开发，Web API 服务开发，CLI 工具开发。

koa是什么？

Koa 是一个基于 Node.js 的 web 框架，主要用于开发 web 应用后端服务和 API 服务。

js vue node koa 这些技术之间的关系

Vue 是一个前端框架，里面的脚本部分可以使用 JavaScript 进行开发；Koa 是一个服务器端web框架，用于给 Node.js 在服务器端的开发赋能，加快开发速度；Node.js 是一个服务器端开发语言，开发时的语法基于 JavaScript。

可以总结为，Node.js 给 JavaScript 插上了飞向各端的翅膀，引来了前端圈的大繁荣，进而诞生了 Vue.js 这样的前端框架，进一步加强了前端的繁荣。
